1 Argument Aggregator {#mainpage}
2 ===================
3 
4 This is the Doxygen documentation for Argument Aggregator, a simple C++11 argument parser.
5 
6 To use just create an argagg::parser object. However, the struct doesn't provide any explicit methods for defining flags. Instead we define the flags using initialization lists.
7 
8  argagg::parser argparser {{
9  {"help", {"-h", "--help"}, "shows this help message", 0},
10  }}
11 
12 A flag is specified by four things: the name of the flag, the strings that activate the flag, the flag's help message, and the number of arguments the flag expects.
13 
14 With the parser defined you actually parse the arguments by calling the argagg::parser::parse() method. If there are any problems then either a std::out_of_range or argagg::unexpected_flag_error exception.
15 
16  argagg::arg args;
17  try {
18  args = argparser.parse(argc, argv);
19  } except (const std::exception& e) {
20  std::cerr << e.what() << std::endl;
21  return EXIT_FAILURE;
22  }
23 
24 You can check if a flag shows up in the arguments using the argagg::args::has_flag() method and giving it the flag name (not the flag text itself). You can write out some simplistic flag help message but streaming the argagg::parser instance itself.
25 
26  if (args.has_flag("help")) {
27  std::cerr << argparser;
28  // -h, --help
29  // shows this help message
30  return EXIT_SUCCESS;
31  }
32 
33 That help message is only for the flags. If you want a usage message it's up to you to provide it.
34 
35  if (args.has_flag("help")) {
36  std::cerr << "Usage: program [options] ARG1 ARG2" << std::endl
37  << argparser;
38  // Usage: program [options] ARG1 ARG2
39  // -h, --help
40  // shows this help message
41  return EXIT_SUCCESS;
42  }
43 
44 Generally argagg tries to do a minimal amount of work to leave most of the control with the user.
45 
46 Finally, you can get all of the positional arguments as an std::vector using the argagg::args::args member.
47
